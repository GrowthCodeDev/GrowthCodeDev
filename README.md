## Engineering Philosophy

> *Simplicity is a feature.*

- Readability is non-negotiable  
- Explicit code beats clever abstractions  
- Small modules scale better than large systems  
- Dependencies are a cost, not a convenience  
- Code should explain **why**, not just **how**  

I aim for software that:
- Remains understandable months later  
- Is easy to test, extend, and remove  
- Fails clearly and predictably  
- Respects performance and resources  

Documentation and structure are part of the code.
