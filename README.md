## Engineering Philosophy

> *Simplicity is a feature.*

- Readability is non-negotiable  
- Explicit > implicit  
- Dependencies are a cost  
- Small modules scale better  
- Code should explain *why*, not just *how*  

I aim for systems that:
- Stay understandable long-term  
- Are easy to extend or delete  
- Fail clearly and predictably  
- Respect performance and resources
